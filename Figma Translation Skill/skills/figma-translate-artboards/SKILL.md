# Figma Translate Artboards

## Workflow

When the user asks to translate a Petlibro Figma artboard into DE, FR, and ES, execute the full readme workflow in this order:

1. Require a Figma design URL that includes `node-id`. Extract both `fileKey` and `nodeId`; convert URL node IDs such as `15836-5009` to `15836:5009`.
2. Load the `figma-use` skill before every Figma `use_figma` call.
3. Extract text from the target node with `scripts/extract-figma-text.js`; replace `__TARGET_NODE_ID__` before calling `use_figma`. The extractor includes instance children because the translation step detaches instances, but returns only `uniqueTexts` by default; set `INCLUDE_TEXT_NODES = true` only when debugging node-level extraction.
4. Save the returned JSON to a temporary file, for example `.tmp/extracted-texts.json`.
5. Run `scripts/build-translation-plan.mjs` to read:
   - the Official Petlibro Glossary Google Sheet,
   - the Active Glossary Google Sheet, auto-discovering and reading every `*_Market` tab such as `Germany_Market` and `France_Market`,
   - `references/Petlibro Localization Prompt v1.2.md`.
6. Use official glossary exact matches directly.
7. Use Active Glossary entries only when `Confidence` is `HIGH`.
8. Send all strings that are not matched by either glossary, plus `MEDIUM`/`LOW` active glossary entries, through AI localization using the generated `.tmp/ai-instruction.md` file. This AI step must follow `references/Petlibro Localization Prompt v1.2.md`, but the `±2` visible-character difference from the EN source is the primary layout constraint for every final translation. Count spaces and line breaks. Within the `±2` limit, choose the most accurate, natural, target-market Petlibro voice.
9. Save the AI response as strict JSON and merge it with the glossary hits by running `scripts/merge-ai-translations.mjs`. Review the merge output's `lengthChecks` before rendering. If any final translation exceeds the `±2` visible-character tolerance, revise it first; do not render by default. This includes glossary-derived text if it would break layout. Do not use awkward abbreviations, English fallback words, vague wording, or grammar errors just to fit. Instead, find the best compact localization that remains truthful and on-brand inside the `±2` limit. If no acceptable option fits after reasonable revision, stop and ask for human/native review or explicit user approval for that exception.
10. Render the final Figma script with `scripts/render-figma-code.mjs`, which injects the target node ID and consolidated translation JSON into `translate.js`.
11. Before calling Figma `use_figma`, preserve UTF-8 exactly. Do not copy rendered scripts or translation JSON from PowerShell `Get-Content` output when they contain accented characters such as `ö`, `ä`, `ü`, `é`, `à`, `ñ`, or `°`; PowerShell may display UTF-8 as mojibake such as `S枚hne`, `R盲tselraten`, `脺berwachung`, or `f眉r`. Pass the script from the UTF-8 file contents, or use the JSON `code` field generated by `--use-figma-code-json`.
12. Run the rendered JavaScript with Figma `use_figma` by passing the complete script source directly as `use_figma.code`. Never use a localhost URL, `fetch`, `XMLHttpRequest`, or a runtime loader inside Figma to load `.tmp/figma-run.js`.
13. Verify the structured `use_figma` result: each `DE`, `FR`, and `ES` clone should be appended, contain translated text, and report `missingFontCount = 0`.
14. Run a mojibake sanity check on the created clones. Search translated text for common corruption characters such as `盲`, `眉`, `脺`, `枚`, `脽`, `茅`, `谩`, `贸`, `铆`, `帽`, `鈥`, and `檌`. If any appear, stop and repair from the original UTF-8 translation JSON before considering the workflow complete.
15. Do not take screenshots during the default workflow. Missing Figma fonts are expected and should be handled by migrating clone text to `Inter`, not by screenshot diagnosis. Use screenshots only when the user explicitly asks for visual QA, or when the structured result reports non-font errors, unexpectedly high untranslated counts, or a layout issue that cannot be diagnosed from node metadata.

Do not use Cursor paths such as `.cursor/skills/...` in Codex. Use this skill directory as the source of `translate.js`.

## Commands

Use the bundled Node scripts from this skill directory:

```powershell
node scripts/build-translation-plan.mjs --texts .tmp/extracted-texts.json --out .tmp/translation-plan.json --ai-instruction-out .tmp/ai-instruction.md --audit-out .tmp/translation-audit.json
node scripts/merge-ai-translations.mjs --plan .tmp/translation-plan.json --ai .tmp/ai-translations.json --out .tmp/consolidated-translations.json
node scripts/render-figma-code.mjs --node-id 24112:853 --translations .tmp/consolidated-translations.json --out .tmp/figma-run.js --use-figma-code-json .tmp/figma-run.use_figma_code.json
```

If Google Sheets access fails because of network sandboxing, rerun the plan command with the required escalation. If Google Sheets permissions block CSV export, ask the user for exported CSV/TSV files and pass them with `--official-csv` and `--active-csv`.

By default, `build-translation-plan.mjs` discovers all `*_Market` tabs in the Active Glossary workbook and merges them before applying `HIGH` confidence entries. Use `--active-csv` or `--active-url` only when intentionally overriding this with a single local export or URL.

For speed and token efficiency, keep the main plan compact by writing the AI instruction and detailed audit trail to `.tmp/ai-instruction.md` and `.tmp/translation-audit.json`. The merge step only needs `.tmp/translation-plan.json` plus the AI response JSON.

## Figma Writeback Transport

The Figma plugin runtime used by `use_figma` does not reliably provide browser networking APIs. Do not serve `.tmp/figma-run.js` over `localhost`, and do not run a loader script that calls `fetch(...)`, `XMLHttpRequest`, dynamic `import(...)`, or any other network/file-loading API inside Figma. This fails in environments where the runtime has no `fetch` or `XMLHttpRequest`.

Instead, pass the rendered JavaScript source itself as the `code` argument of the Figma `use_figma` tool:

1. Render the script with `scripts/render-figma-code.mjs`.
2. If the agent needs a JSON-safe transfer format, use `--use-figma-code-json .tmp/figma-run.use_figma_code.json`.
3. Read that JSON with Node, take its `code` field, and pass that exact string directly to `use_figma.code`.
4. Never copy localized script text from PowerShell display output; use Node UTF-8 file reads or the JSON `code` field.

Valid writeback means the `use_figma` tool receives the complete rendered script as its `code` parameter. A local server, localhost URL, or runtime fetch loader is not a valid workaround.

## Translation Sources

The workflow uses these sources in priority order:

1. Official Petlibro Glossary: exact EN matches for product names, slogans, and one-sentence descriptions.
2. Active Glossary: exact EN matches with `HIGH` confidence only.
3. AI localization: strings unmatched by both glossary sources, plus Active Glossary `MEDIUM`/`LOW` entries, using `references/Petlibro Localization Prompt v1.2.md`; the `±2` visible-character difference from the EN source is a hard pre-render constraint for the final consolidated translations unless the user explicitly approves an exception.
4. `translate.js` runtime template: contains no cached translations; it only applies the injected consolidated translation JSON inside Figma.

Reference files:

- `references/Petlibro Glossary Rule.md` describes the official glossary columns.
- `references/Petlibro Glossary(Active) Rule.md` describes active glossary confidence handling.
- `references/Petlibro Localization Prompt v1.2.md` defines the Petlibro tone and product-name rules.

Keep Petlibro product names in English unless an approved glossary explicitly says otherwise.

## Script Behavior

1. Finds the original artboard by ID.
2. Renames the original artboard to `EN`.
3. Clones the artboard three times, naming them `DE`, `FR`, and `ES`.
4. Places the clones exactly on the same Figma `PAGE` as the original, spaced 100px apart to the right.
5. Replaces `Söhne` with `Inter` and maps `Leicht` to `Light`, `Buch` to `Regular`, and `Kräftig` to `Semi Bold`, including mixed text ranges.
6. Translates the English text into German, French, and Spanish using only the injected consolidated translations.

## Encoding and Font Bug Prevention

This workflow has previously produced mojibake in Figma when a rendered UTF-8 script was copied from PowerShell display output instead of being passed as UTF-8 source bytes. Examples of the bad output include `Mehr Kontrolle. Weniger R盲tselraten.` and `24/7-脺berwachung f眉r mehr Ruhe`.

To prevent this:

- Treat the files on disk as UTF-8 and prefer Node-based file reads/writes for generated scripts and translation JSON.
- Do not trust terminal display output as the source of truth for non-ASCII text. PowerShell may display valid UTF-8 incorrectly even when the file bytes are correct.
- If a rendered Figma script contains localized strings, pass the exact file contents to `use_figma`; if the transport path risks re-encoding, wrap the script or translation payload in base64 or JSON escaping generated from Node.
- Keep the font mapping literals as real UTF-8: `Söhne`, `Leicht`, `Buch`, and `Kräftig`.
- The required font mapping is exactly: `Söhne Leicht -> Inter Light`, `Söhne Buch -> Inter Regular`, and `Söhne Kräftig -> Inter Semi Bold`.
- For mixed-font text nodes, do not collapse the whole node to one Inter style. Preserve local emphasis ranges after translation: numeric/name segments originally in `Söhne Kräftig` must remain `Inter Semi Bold`, while surrounding `Söhne Buch` text becomes `Inter Regular`.
- After Figma execution, verify `missingFontCount = 0`, `mojibakeCount = 0`, and `over2LengthDeltaCount = 0` for every clone before reporting success, unless the user explicitly approved a recorded length exception. Also report `maxAbsDelta`.
- Length QA must compare every translated text node against the corresponding EN text node by traversal order. Compute character counts with JavaScript spread syntax (`[...text].length`) so accented characters count as single visible characters. The `±2` visible-character difference has higher priority than preserving a longer literal translation: satisfy the length limit first, then choose the best accurate, warm, casual, smart, and natural Petlibro localization within that space.
- Unified `normalizeText` logic must be used across all scripts to prevent duplicate translations caused by punctuation or whitespace variance.
- The font mapping is a separate hard requirement: `Söhne Leicht -> Inter Light`, `Söhne Buch -> Inter Regular`, and `Söhne Kräftig -> Inter Semi Bold`.

## Handling Errors

- If the target node is missing, stop and ask for a node-specific Figma URL.
- If text nodes are locked inside instances, the script detaches instances in the clone before writing text.
- If `Söhne` is missing in the Figma MCP environment, the script converts clone text to `Inter` before appending the clone to the page.
- If the final result still has untranslated strings, update the AI translations JSON or glossary source and rerun from the merge/render step.
